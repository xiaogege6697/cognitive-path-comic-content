# 行业模块模板

新增行业时建立独立文件，至少定义：

```yaml
module_name: 行业名称
audience: 典型用户
high_frequency_questions: 高频真实问题
professional_framework: 行业判断框架
required_sources: 优先的一手来源
time_sensitivity: 哪些信息会变化
evidence_visuals: 可视化事实
case_fields: 行业特殊字段
action_patterns: 可执行建议类型
scope_limits: 适用范围
compliance: 不得说满或越界的内容
red_flags: 必须升级专业帮助的情形
```

行业模块只提供专业判断，不改变Core的数据流、每页单一认知动作和1—2格限制。
