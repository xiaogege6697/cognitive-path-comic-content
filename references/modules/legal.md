# 法律模块

## 适用方式

法律内容的目标是教用户识别法律关系、证据和下一步，不代替个案法律意见。先确认法域与当前日期；法律、司法解释和平台规则可能变化，优先核对官方一手来源。

## EpisodeBrief增补字段

```yaml
jurisdiction: 中国大陆或其他法域
event_timeline: 关键时间线
parties: 当事人关系与身份
claims: 各方分别主张什么
legal_relationships: 可能涉及的法律关系
disputed_issues: 真正争议焦点
burden_of_proof: 谁需要证明什么
available_evidence: 已有证据
missing_evidence: 缺失证据
authority:
  - title: 法律或官方来源
    effective_date: 生效或更新时间
    url: 官方链接
exceptions: 条件与例外
next_steps: 协商、取证、咨询、诉讼等合法路径
```

## 判断顺序

1. 确定法域、时间和当事人关系。
2. 把口语问题翻译成一个或多个法律关系。
3. 分离各方主张，不替任何一方补事实。
4. 找出争议焦点与举证责任。
5. 核对现行法律和官方案例，不用网络段子当依据。
6. 将结论写成“条件 → 判断 → 例外”，避免“一定、都、只要”。
7. 给合法、可执行且不破坏证据的下一步。

## 视觉证据

聊天、转账、合同、借条、工资条、账单和时间线只作为“证据类型示意”。不得把虚构截图包装成真实案件材料。画面中金额、日期、备注和人物陈述必须前后一致。

## 禁止事项

- 不包赢、不预测确定判决。
- 不伪造法条编号、裁判文号或官方来源。
- 不把金额大小、特殊数字、性别或身份当成单一结论。
- 不鼓励删记录、转移财产、诱导取证、偷录隐私或对抗法院。
- 不泄露真实姓名、电话、地址、身份证、账号等隐私。
- 遇到威胁、暴力、刑事风险或诉讼期限问题，提示及时寻求当地专业帮助。

## 法律页结论句式

- “通常……，但仍需结合……判断。”
- “关键不在A，而在能否证明B。”
- “这项事实是判断线索，不是单独定案标准。”
- “可能存在多种法律性质，应逐笔／逐项区分。”
